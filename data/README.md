# Data Layout

This repository keeps the dataset folder under version control as a structure, not as a storage bucket.

## Folders

- `raw/metadata/`: original metadata tables such as `MAAD_Face.csv`
- `raw/downloads/`: source archives downloaded from remote storage
- `processed/manifests/`: balanced manifests used for training and evaluation
- `processed/archives/`: filtered archives derived from the raw source dataset
- `reports/`: JSON artifacts generated by the data ingestion CLI

## Recommended workflow

1. Download the source archive with `python3 scripts/data.py download --config config/data/maad_face.yaml --url ...`
2. Build the balanced manifest with `python3 scripts/data.py build-manifest`
3. Create the filtered archive with `python3 scripts/data.py filter-archive`

Large files are intentionally ignored in `.gitignore`, while the folder structure stays in the repository.
