# Data Layout

This repository keeps the `data/` directory versioned as a structure, not as a bucket of large files.

## Sources

- MAAD-Face metadata and annotations: <https://github.com/pterhoer/MAAD-Face>
- Local image archive used by this workflow: <https://www.kaggle.com/datasets/hearfool/vggface2>

## Folders

- `raw/metadata/`: original metadata tables such as `MAAD_Face.csv`
- `raw/downloads/`: manually downloaded source archives such as `vggface2.zip`
- `processed/manifests/`: balanced manifests used by training and evaluation
- `processed/archives/`: filtered archives derived from the source image ZIP
- `reports/`: JSON artifacts generated by the data ingestion CLI

## Recommended workflow

1. Place `MAAD_Face.csv` under `data/raw/metadata/`
2. Place the Kaggle-downloaded `vggface2.zip` under `data/raw/downloads/`
3. Run `python3 scripts/data.py build-manifest --config config/data/maad_face.yaml`
4. Run `python3 scripts/data.py filter-archive --config config/data/maad_face.yaml`

Large files are intentionally ignored in `.gitignore`, while the folder structure and operational metadata stay in the repository.
